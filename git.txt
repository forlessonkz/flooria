git add "file name" // Команда git add используется в Git для добавления изменений в индекс (или «staging area»).
            Это промежуточная область, где изменения подготавливаются перед коммитом в репозиторий.
            Когда вы выполняете команду git add, Git сохраняет текущую версию изменённых файлов в индексе, 
            чтобы они были включены в следующий коммит. Эта команда позволяет вам контролировать, 
            какие изменения будут сохранены в истории вашего репозитория.

git commit // Команда git commit используется в системе контроля версий Git для сохранения изменений в локальном репозитории.
            При выполнении команды git commit, Git сохраняет текущее состояние индексированных (staged) файлов в виде коммита (commit). 
            Коммит содержит метаданные, такие как автор изменений, дата и время коммита, а также сообщение, описывающее изменения.

git log // Выводит информацию о всех коммитах.

git push // Команда git push используется в системе контроля версий Git 
            для отправки изменений из локального репозитория в удаленный репозиторий.
